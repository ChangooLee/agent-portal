# 개발자 선호도 기록

Agent Portal 개발에서 사용자가 선호하거나 싫어하는 패턴을 기록합니다.

## 형식

```markdown
## YYYY-MM-DD: 선호도 제목

**상황**: 선호도 발생 상황
**피드백**: ✅ 잘 잡았어 / ❌ 이거 싫어함
**이유**: 피드백 이유
**향후 적용**: 다음 유사 결정 시 적용 방법

---
```

## 2025-11-19: 새 채팅 버튼 새로고침 문제

**상황**: 사이드바의 "새 채팅" 버튼을 누를 때 항상 페이지 전체 새로고침 발생
**피드백**: ❌ 이거 싫어함
**이유**: 전체 페이지 새로고침은 UX 저하, 상태 초기화, 불필요한 네트워크 요청 발생
**해결**: `window.location.href` → SvelteKit `goto()` 사용
**향후 적용**: 
- 모든 내부 네비게이션에 `goto()` 사용
- `window.location.href`는 외부 URL만 사용
- SPA 네비게이션으로 부드러운 사용자 경험 제공

## 2025-11-19: 문서 중복 방지 가이드 추가

**상황**: 신규 문서가 계속 추가되어 문서 관리 부담 증가
**피드백**: ❌ 이거 싫어함
**이유**: 기존 문서를 제대로 검토하지 않고 신규 문서를 추가하면 중복 내용이 쌓이고, 어떤 문서를 참조해야 할지 혼란스러워짐
**해결**: 
- 모든 주요 문서에 "문서 생성 전 검토" 가이드 추가
- `.cursorrules`, `.cursor/rules/*.mdc`, `CLAUDE.md`, `AGENTS.md`에 검색 절차 명시
- `docs/DOCUMENT_MANAGEMENT.md` 신규 생성하여 전용 가이드 제공
**향후 적용**: 
- 문서 생성 전 반드시 `grep -r "키워드" .cursor/learnings/ docs/ .cursor/rules/` 실행
- 통합 가능하면 기존 문서에 섹션 추가 우선
- 신규 생성은 완전히 새로운 주제일 때만
- 임시 문서(`TEMP_*.md`)는 작업 완료 후 정리
