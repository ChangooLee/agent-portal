# 문서 관리 가이드

## 문서 디렉토리 구조

```
agent-portal/
├── .cursor/
│   ├── rules/          # 개발 규칙 (.mdc)
│   └── learnings/      # 학습 내용 (.md)
├── docs/               # 상세 가이드 (.md)
├── AGENTS.md           # 에이전트 워크플로우
├── CLAUDE.md           # 프로젝트 가이드
├── README.md           # 프로젝트 개요
└── *.md                # 기타 프로젝트 문서
```

## 신규 문서 생성 프로세스

### 1단계: 기존 문서 검색 (필수)
```bash
# 키워드로 전체 검색
grep -r "키워드" .cursor/ docs/ *.md

# 파일명으로 검색
find . -name "*키워드*.md" -o -name "*키워드*.mdc"
```

### 2단계: 통합 가능성 판단

**통합하는 경우** (우선):
- 기존 문서에 관련 내용 존재
- 같은 카테고리/주제
- 섹션 추가로 충분

**신규 생성하는 경우**:
- 완전히 새로운 주제
- 대규모 독립 작업
- 특정 기술/도구 전용

### 3단계: 적절한 위치 선택

| 내용 유형 | 위치 | 파일명 규칙 |
|-----------|------|------------|
| 학습 내용 | `.cursor/learnings/` | `{category}.md` |
| 개발 규칙 | `.cursor/rules/` | `{feature}.mdc` |
| 상세 가이드 | `docs/` | `{TOPIC}_*.md` |
| 임시 문서 | 루트 | `TEMP_*.md` 또는 `IMPLEMENTATION_*.md` |

### 4단계: 파일명 규칙

**학습 내용** (`.cursor/learnings/`):
- `ui-patterns.md` - UI 개발 패턴
- `api-patterns.md` - API 개발 패턴
- `bug-fixes.md` - 버그 수정 사례
- `{tool}-integration.md` - 특정 도구 통합 (예: `litellm-integration.md`)

**개발 규칙** (`.cursor/rules/`):
- `{feature}-{type}.mdc` (예: `ui-development.mdc`)
- 하이픈으로 단어 구분 (kebab-case)

**상세 가이드** (`docs/`):
- `{TOPIC}_*.md` - 대문자 시작 (예: `LITELLM_SETUP.md`)
- 언더스코어로 단어 구분

**임시 문서** (루트):
- `TEMP_*.md` - 임시 메모
- `IMPLEMENTATION_*.md` - 구현 분석
- 작업 완료 후 `.backup/temp-docs/`로 이동 또는 삭제

## 문서 통합 가이드

### 기존 문서에 추가하는 방법

1. **날짜 섹션 추가**:
```markdown
## 2025-11-19: 새로운 학습 내용

**요청**: ...
**적용**: ...
**피드백**: ...

---
```

2. **카테고리 섹션 추가**:
```markdown
### 새로운 패턴

**설명**: ...
**사용법**: ...
```

### 문서 정리 주기

**주간 리뷰** (매주 금요일):
- 임시 문서 정리
- 중복 내용 통합
- 오래된 내용 아카이브

**월간 리뷰** (매월 첫째 주):
- 문서 구조 재정비
- 카테고리 재분류
- 핵심 문서 업데이트
